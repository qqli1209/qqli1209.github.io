href 和 src 的区别

发表于 2017-03-04 浏览 - 次 分类于 前端学习 | HTML

href 和 src

写代码的时候经常想好久link,a,script之类的到底是用 href 还是 src，这两个属性给我的感觉差不多。曾经有一段时间通过死记硬背来记忆，link和a用的是href，script，iframe，img用的是 src。但平时还是经常弄混，用的时候心里没底。决定仔细推敲一下，方便记忆。毕竟理解性的记忆最深刻。

href

href 是 hypertext reference，即超文本引用，用来建立当前元素和文档之间的链接。例如：

<link rel="stylesheet" type="text/css" href="style.css">

浏览器明白当前资源是一个样式表，页面解析不会暂停。相似地，对于a元素也是使用 href。而使用@import加载 css 时，则没有这样的特性。因此建议使用link而非@import加载 css 资源。

src

src 是 source ，即嵌入当前资源到当前文档定义的位置，例如：

<script type="text/javascript" src="test.js"></script>

当浏览器解析到上面的代码时，在浏览器下载，编译，执行这个文件之前，页面的加载和处理会被暂停。相似地，对于iframe和img 用的是src。对于img，浏览器会停止加载直到完成提取和加载图像。这也是建议把 JavaScript 文件放到底部加载的原因。
